3、上下文拼接
提示词设计分为三层：
边界层只写“禁止”和“底线”，告诉模型不能做什么
决策层写逻辑，用过程而不是结果来描述，避免使用"聪明地"、"合理地"这种模糊的副词
回复层写失败时的退化策略，告诉模型出错了怎么办

提示词要以 Prefix Cache 为核心来设计，参考Claude Code的四层提示词设计思路

工具输出会导致上下文暴涨，一个较好的做法：工具输出截断显示，完整结果存盘保存，提供查询链接

超限自动触发压缩（达到上下文窗口的80%？），依然要保持 Prefix Cache，参考 Claude Code 的做法
关键信息先提取出来，不做压缩，避免模型遗忘最初的目标

@file 功能只插入 system_reminder 提醒，不直接注入文件内容，避免上下文暴涨
